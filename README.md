# ezshell

This is my solution to the need for a practical desktop shell for Hyprland that does everything you need it to but never gets in the way.

# Dependencies

**ezshell** is developed on NixOS and its flake is the recommended installation method. However here are the (untested) dependencies on Arch Linux, you can figure it out from there.

### Required

```
aylurs-gtk-shell-git
libadwaita
libsoup3
glib-networking
accountsservice
papirus-icon-theme
libmanette
libgudev
libvirt-glib
gjs
wlr-randr
imagemagick
```

Node.js/NPM is also required for installing dependencies, recommended version 24.

### Optional

**ezshell** tries to keep dependencies optional when possible, but these packages enable additional features.

```
brightnessctl
libqalculate
mozlz4a
qrencode
lm_sensors
```

# Configuration

**ezshell** is configured using a yaml file, usually at `~/.config/ezshell/config.yaml`. Most of the configuration is optional.

#### Example Config

```yaml
theme:
  wallpaperDirectory: /home/eyezah/wallpapers
  mutableWallpaperDirectory: /home/eyezah/wallpapers
  defaultWallustTheme: ashes-dark
  wallustThemeFile: /home/eyezah/.cache/ags/wallust.scss
greeter:
  sessionsDirectory: /etc/example-desktop-sessions
  monitors:
    eDP-1:
      enabled: true
      scale: 2
      mode: 2560x1600@60
  cursorTheme: Posy
launcher:
  code:
    command: codium
    icon: vscodium-symbolic
stats:
  slskd:
    address: http://127.0.0.1:5030
    apiKey: bDkRV9MCUuTZ6idBpc3F6DRS
  sensors:
    - adapter: macsmc_hwmon-isa-0000
      sensor: Fan
      icon: sensors-fan-symbolic
      label: "{RAW} RPM"
    - adapter: macsmc_hwmon-isa-0000
      sensor: Total System Power
      icon: power-symbolic
      label: "{SIMPLIFY}W"
      min: 0
      max: 35
  jellyfin:
    address: http://127.0.0.1:8096
    apiKey: teJ5hYVpwo6owiF58QbbAfgB
dock:
  enable: true
```

Most options are fairly self explanatory, but some have niche use cases:

#### theme.mutableWallpaperDirectory

In some immutable distros like NixOS, some configurations will include the wallpaper directory, making an immutable copy of it. In this instance, `theme.wallpaperDirectory` should be pointed at the immutable copy and `theme.mutableWallpaperDirectory` to the original copy in the system configuration directory. If you don't use an immutable distro like NixOS, or you don't include your wallpaper directory in your system configuration, you should omit this option.

#### theme.wallustThemeFile

**ezshell** expects this option to point to the location of a `.scss` file automatically generated by Wallust. The Wallust template file is as follows:

```yaml
$wallust-background: {{background}};
$wallust-foreground: {{foreground}};
$wallust-cursor: {{cursor}};

$wallust-color0: {{color0}};
$wallust-color1: {{color1}};
$wallust-color2: {{color2}};
$wallust-color3: {{color3}};
$wallust-color4: {{color4}};
$wallust-color5: {{color5}};
$wallust-color6: {{color6}};
$wallust-color7: {{color7}};
$wallust-color8: {{color8}};
$wallust-color9: {{color9}};
$wallust-color10: {{color10}};
$wallust-color11: {{color11}};
$wallust-color12: {{color12}};
$wallust-color13: {{color13}};
$wallust-color14: {{color14}};
$wallust-color15: {{color15}};
```

#### stats.sensors

This module pulls from the `lm_sensors` package. If you have it installed, run `sensors` to see what metrics you have available. If you don't specify `min` and `max` for a metric, **ezshell** will do it's best to figure it out for you.

#### greeter.sessionsDirectory

**ezshell** expects this to be the path to a directory containing `.desktop` files for starting desktop sessions.

## NixOS

**ezshell** can be imported easily into a NixOS flake:

```nix
eyezah-ui.url = "github:eyezahhhh/eyezah-ui";
```

Configuration is done using Home-Manager:

```nix
{ inputs, ... }: {
  imports = [
    inputs.eyezah-ui.homeManagerModules.default
  ];

  programs.ezshell = {
    shell.enable = true;
    greeter.enable = true;
    instanceId = "ezshell"; # I don't know why you'd want to change this, but you can!

    exposeAgs = true; # Expose the underlying AGS binary in order to send commands to ezshell

    settings = {
      # YAML file options are configured here instead, for example:
      launcher = {
        code = {
          command = lib.getExe pkgs.vscodium; # Import "lib" and "pkgs" too if you want this to work
          icon = "vscodium-symbolic";
        };
      };
    };
  };
}
```
